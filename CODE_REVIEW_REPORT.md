# Oh-My-Claw 代码审查报告（2026 刷新版）

## 总结

本次审查目标从“基础可用”提升到“复杂任务稳定执行”。
近期主要改动聚焦：MiniMax 兼容、上下文压缩、心跳可观测、记忆检索与 CLI 体验。

## 已修复的关键问题

1. **MiniMax 多 system 导致 2013 错误**
   - 处理：首条 system 保留，后续 system 降级为 user note。

2. **MiniMax 工具协议文本化返回未被解析**
   - 处理：新增 `<minimax:tool_call><invoke>` 解析，恢复结构化 ToolCall。

3. **复杂任务下工具历史协议错位风险**
   - 处理：最近 2 轮保留结构化，旧轮文本化，减少协议校验失败。

4. **长任务缺少过程可观测**
   - 处理：心跳 + 看门狗 + `/heartbeat` 命令。

## 仍需持续优化的点

1. **大内容写文件稳定性**
   - 问题：长内容易触发空参数或截断。
   - 建议：落地强制分块写入策略并增加回归测试。

2. **复杂任务策略控制**
   - 问题：规划、技能探索、shell 探测可能拉长链路。
   - 建议：按任务类型限制工具集，降低无关探索。

3. **文档与代码同步机制**
   - 问题：历史文档容易过时。
   - 建议：发布前执行“文档一致性检查清单”。

## 当前结论

工程已从“单轮演示型”迈向“多轮工程执行型”，建议下一阶段重点做端到端稳定性与发布流程标准化。
