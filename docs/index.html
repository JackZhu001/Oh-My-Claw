Oh-My-Claw

一只真正“会干活”的工程猫爪助手
基于 Function Calling 构建，能读仓库、跑工具、盯流程，把复杂任务推进到落地

项目概览

🎯

项目目标

把“看懂仓库、找准问题、推进改造”变成一条顺滑链路，不让你在大项目里迷路，也不让任务卡在嘴上。

⚡

核心特性

原生 Function Calling、模块化工具系统、Pydantic 参数验证、团队协作运行时、可追溯日志与 Token 统计。

🏗️

技术架构

Python 3.10+ + MiniMax API + Rich CLI + RepoRAG，面向真实工程任务而不是“演示级问答”。

核心能力

读取和理解代码文件、分析项目结构、识别关键模块和依赖关系

02

搜索特定的代码模式、查找函数/类定义、全文搜索代码内容

重构建议

代码优化建议、结构改进方案、性能优化提示

05

任务规划

06

子代理协作

支持任务委托、代码探索、计划生成、信息总结

架构与流程

Oh-My-Claw 采用基于 Function Calling 的 Agent 循环架构，与传统 ReAct 范式相比，LLM 直接返回结构化的 tool_calls，更加可靠。

步骤 1: 接收用户输入

用户通过 CLI 提出问题或任务

↓

步骤 2: 调用 LLM

带上工具列表，让 LLM 决定是否需要调用工具

↓

步骤 3: 执行工具

如果 LLM 请求调用工具，执行工具并获取结果

↓

步骤 4: 返回结果

将工具结果返回给 LLM，重复步骤 2-4

↓

步骤 5: 给出答案

LLM 给出最终答案，或达到最大轮数限制

核心模块

🤖 Agent 核心

实现基于 Function Calling 的 Agent 循环，处理用户输入、调用 LLM、执行工具、返回结果

codemate_agent/agent/agent.py

🧠 LLM 客户端

与 MiniMax API 通信，管理 API 密钥、处理 Function Calling 请求、Token 统计

codemate_agent/llm/client.py

🔧 工具系统

模块化工具管理，支持文件、搜索、Shell、Todo、Task 等多种工具类型

codemate_agent/tools/registry.py

📊 数据模型

定义 Message、ToolCall、LLMResponse、TokenUsage 等核心数据结构

codemate_agent/schema.py

📝 日志系统

三层日志架构：运行时日志、Trace 轨迹日志、Metrics 统计

codemate_agent/logging/

💻 CLI 入口

命令行交互入口，支持交互模式和单次查询模式

codemate_agent/cli.py

💓 心跳机制

长任务可观测、超时可告警，避免静默失败。支持阶段心跳上报、后台周期唤醒检查待办、超时自动告警。

codemate_agent/agent/loop_detector.py

📋 任务规划器

检测复杂任务并生成执行计划，自动分解任务、跟踪进度

codemate_agent/planner/planner.py

💾 持久化系统

会话存储、记忆管理、会话索引，支持对话历史的持久化

codemate_agent/persistence/

🗜️ 上下文压缩

处理长对话的上下文压缩，检测长度、压缩历史、截断输出

codemate_agent/context/compressor.py

🎯 Skill 系统

支持预定义的专业任务流程，如代码审查等

codemate_agent/skill/

👥 子代理系统

支持任务委托给子代理，包括通用、探索、规划、总结等类型

codemate_agent/subagent/

✅ 验证模块

统一验证工具调用参数，类型验证、值检查、使用提示

codemate_agent/validation/argument_validator.py

注意事项

🔑

API 密钥配置

使用前需要在 .env 文件中配置 API_KEY，确保有足够的 API 配额。

⚠️

危险操作确认

delete_file、write_file、run_shell 等危险工具需要用户确认，支持批量确认模式。

📏

上下文长度限制

启用上下文压缩功能，当对话历史过长时自动压缩，避免超出 Token 限制。

🔄

循环检测机制

内置循环检测器，当 Agent 陷入循环时自动中断，避免无限执行。

📊

日志与监控

完整的日志系统，支持 Trace 轨迹回放和 Metrics 统计，便于调试和优化。

🐍

Python 版本要求

需要 Python 3.10 或更高版本，建议使用 conda 或 venv 管理虚拟环境。
